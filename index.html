How it works

  1. CloudFormation provisions a CloudWatch Dashboard, Alarms, and an SNS Topic.
  2. Metrics are collected from the EC2 instance (no agent required).
  3. Alarms trigger when thresholds are breached and send SNS emails.
  4. Dashboard provides a single-pane view for CPU, network, status checks, and credits.
# Deploy (example)
aws cloudformation deploy \
  --template-file cloudformation/ec2-monitoring.yaml \
  --stack-name ec2-monitoring-stack \
  --capabilities CAPABILITY_NAMED_IAM \
  --region ap-south-1 \
  --parameter-overrides InstanceId=i-xxxxxxxx AlertEmail=you@example.com
